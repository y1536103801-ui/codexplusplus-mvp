# 2026-07-12 测试与部署记录

## 1. ChatGPT session 转短期授权

状态：真实链路在 ChatGPT session 获取前受阻。

使用 Google 账号登录 ChatGPT 时，Google OAuth 已进入 OpenAI 登录流程，但随后要求电话号码验证。当前测试账号没有可用电话号码，无法完成登录，因此没有取得真实 ChatGPT session JSON，尚不能宣称“ChatGPT session → 最小 auth.json → 短期授权入池”真实链路通过。

该限制只适用于本次测试账号和登录环境。后端已有自动化测试覆盖 session JSON 解析、敏感资料裁剪、最小 auth.json 生成、短期授权标记、过期时间选择、同账号更新和内部 API Key 复用；这些代码测试不能替代真实会话链路测试。

后续条件：提供一个能够正常完成 ChatGPT 登录且不会停在电话号码验证的账号，由账号持有人在可见登录页自行完成密码、验证码和其他身份验证。

## 3. Windows 客户端验证

状态：本地后端基线测试通过；免费服务器部署完成后仍需切换服务器地址复测。

验证范围包括客户端登录、Codex 桌面端检测、一键启动、真实请求、后端号池归属、Token 扣费、运行心跳、停止租约释放和本机配置恢复。测试只能通过正式客户端和 Codex 桌面应用完成，不使用外部脚本绕过产品流程。

本次通过正式客户端点击“启动 Codex”，官方 Codex 桌面应用成功返回 `VM-POOL-OK`。后端记录用户 1 使用上游账号 `up_180`、模型 `gpt-5.6-sol`、实际用量 20259 token；运行期间号池显示用户 1 正在使用，停止后当前使用者清空并保留最近使用者 1，本机 Codex 账号恢复为“未识别”。

测试发现并修复一个并发计费问题：官方桌面应用同时发起主请求和辅助请求时，辅助请求的预留额度曾错误限制主请求最终扣费。主请求实际用量 20259 token，但首次只扣 15633 token，且账本错误标记为“余额扣至 0”。后端现改为完成请求按真实用量扣费，最多扣到用户物理余额；其他请求的预留额度只控制新请求准入。已增加并发回归测试，历史测试差额 4626 token 通过管理员调整接口校正并写入审计，用户 1 最终余额为 10669 token。
