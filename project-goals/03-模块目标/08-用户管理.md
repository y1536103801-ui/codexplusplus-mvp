用户管理
========

目标
----

让管理员管理普通用户。

必须做到
--------

1，创建普通用户。
2，停用和启用普通用户。
3，查看普通用户状态。
4，管理普通用户的 Codex+++ token 余额、最近充值状态和可用状态。
5，关键操作写入审计记录。
6，管理员创建普通用户时可以设置普通用户密码。
7，普通用户密码内容由管理员决定，系统不得强制非空、长度、复杂度或安全强度提醒。
8，用户列表显示用户账号、token 余额、最近充值状态、账号状态和最近使用时间。
9，用户列表需要分页。
10，用户列表支持搜索账号。
11，管理员创建普通用户时不要求设置初始 token；未设置时初始 token 余额为 0。
12，管理员允许手动调整普通用户 token 余额。
13，管理员手动调整 token 必须形成普通用户可见的 token 收入或消耗流水。
14，用户被停用后，历史充值记录仍保留。
15，管理员手动调整普通用户 token 时不强制填写备注；未填写备注时，普通用户流水来源显示为“管理员调整”。
16，管理员可以在普通用户详情中重置普通用户密码。
17，重置普通用户密码不得设置非空、长度、复杂度或安全强度限制。
18，管理员创建普通用户时不展示一次性初始密码；密码由管理员设置，保存后不再显示原文。
19，管理员手动充值或调整 token 放在普通用户详情页。

不得做到
--------

1，提供普通用户自助注册。
2，提供普通用户浏览器账户页面。
3，允许普通用户自行管理账户凭据。
4，阻止管理员为普通用户设置空密码、短密码或任意自选密码。
5，因用户停用而删除或撤销历史充值记录。
6，在创建普通用户后展示一次性初始密码。
7，把手动充值或调整 token 放在待确认充值页面。
