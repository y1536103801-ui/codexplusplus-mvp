普通用户高级模式
================

目标
----

让普通用户在需要时查看更多记录和状态，但不进入技术配置。

必须做到
--------

1，只能在登录后进入。
2，不作为默认界面，只能由简洁模式中的高级模式按钮进入。
3，展示账号、最近充值状态、Codex+++ token 余额、用量、token 收入和消耗流水、历史充值请求、设备和安全状态。
4，提供返回简洁模式的动作。
5，所有文案使用中文。
6，展示 token 用量明细。
7，token 用量明细只显示时间、模型、input token、cached input token、output token 和总 token。
8，token 用量明细需要分页，每页默认显示 10 条。
9，历史充值请求需要分页，每页默认显示 10 条。
10，历史充值请求只显示 Token 充值项、状态、提交时间和确认时间。
11，token 用量明细列表可用小型横向堆叠条表达 input token、cached input token 和 output token 的组成，但数字字段仍是主信息。
12，token 收入和消耗流水采用最近 30 日正负柱状图加流水表。
13，最近 30 日正负柱状图中，token 收入向上显示，token 消耗向下显示。
14，流水表字段为时间、类型、token 变动数量、变动后余额和来源。
15，脱敏诊断首版只显示本机 Codex 是否检测到、本机 Codex 是否正在运行、本机 Codex 版本和最近一次启动失败原因。
16，客户端更新区提供检查更新和静默更新动作；更新进行时按钮显示“正在更新”并禁止重复触发。

不得展示
--------

1，账户凭据。
2，API Key 原文。
3，proxy。
4，endpoint。
5，base_url。
6，内部 gateway 配置。
7，诊断原文。
8，确认管理员。
9，模型选择或技术服务档位。
10，路径细节、API Key、proxy、endpoint、base_url 或内部 gateway 配置。
