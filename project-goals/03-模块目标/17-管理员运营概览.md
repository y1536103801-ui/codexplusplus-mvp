管理员运营概览
==============

目标
----

让管理员进入 Web 管理后台后先看到运营状态。

必须做到
--------

1，显示待确认充值数量。
2，显示可用号池账号数量。
3，显示今日 token 用量。
4，概览中的数字可以点击进入对应管理页面。
5，首版不显示异常数量。
6，运营概览只用于提示待处理充值、可供调度资源和当天消耗规模，不作为财务报表。
7，今日 token 用量按 Codex 官方 daily usage bucket 口径统计，采用 UTC 自然日，不采用最近 24 小时滚动窗口。
8，需要提供单独的待确认充值页面；概览中的待确认充值数量点击后进入该页面。
9，可用号池账号数量点击后进入号池账号列表，并自动筛选为可用账号。

不得做到
--------

1，把运营概览做成普通用户页面。
2，展示 Codex 账号凭据原文。
3，展示 API Key 原文。
4，引入复杂财务、发票、税费、催收或多租户报表。
5，在未定义清楚异常类型前展示异常数量。
6，批量确认充值请求。
