状态与错误
==========

目标
----

让客户端、管理后台和后端使用同一套状态和错误语义。

必须做到
--------

1，区分 Codex+++ token 余额不足。
2，区分 Codex 账号余额不足。
3，区分 API Key 不可用。
4，区分 Token 充值项不可用。
5，区分设备不可用。
6，普通用户错误文案使用中文。
7，管理员可看到可审计、可排障的内部状态。
8，普通用户账号停用导致登录失败时，不显示“账号不可用”。
9，设备停用导致登录失败时，不显示“设备不可用”。
10，客户端本机启动失败时，只向普通用户显示中文提示和下一步动作。
11，管理员后台只展示运营和资源状态，不展示每个普通用户本机启动过程。
12，Codex+++ token 余额不足时，客户端不额外显示“余额不足”提示；后端不再提供后续 Codex 账号、API Key 或等价可用路由。

不得做到
--------

1，用同一个错误文案同时表示 Codex+++ token 余额不足和 Codex 账号余额不足。
2，把内部错误原文展示给普通用户。
3，把 API Key、账户凭据或内部路由暴露给普通用户。
4，把普通用户本机启动失败做成管理员后台实时监控。
