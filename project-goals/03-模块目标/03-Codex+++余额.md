Codex+++ 余额
=============

目标
----

让普通用户知道自己在 Codex+++ 内还能使用多少服务。

必须做到
--------

1，显示剩余 token 精确数字。
2，显示最近充值状态。
3，在 Codex+++ token 余额充足时允许启动 Codex。
4，在 Codex+++ token 余额不足时不主动触发充值、联系管理员或 API Key 切换。
5，Codex+++ token 余额不足不得触发 API Key 自动切换。
6，Codex+++ 余额首版以 token 表达。
7，Codex+++ token 余额不足时，后端不再提供后续 Codex 账号、API Key 或等价可用路由。
8，Codex+++ token 余额为 0 时仍允许普通用户登录客户端。
9，简洁模式中的剩余 token 不使用“充足 / 不足”状态词替代数字。

不得混淆
--------

1，Codex+++ token 余额不是 Codex 账号余额。
2，Codex+++ token 余额不足不是上游账号不可用。
3，Codex+++ token 余额不足不是切换 API Key 的原因。
4，Codex+++ token 余额不足不是客户端弹出充值引导、自动联系管理员或自动发起充值的原因。
5，客户端不得额外显示“余额不足”提示；Codex 自身负责呈现不能继续使用的提示。
6，余额为 0 不是登录失败原因。
