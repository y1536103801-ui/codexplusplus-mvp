官网与账号购买
==============

模块目标
--------

官网是 Codex+++ 的公开销售入口，用于介绍项目、分别提供 Windows 与 macOS 客户端下载、展示当前可售 Token 套餐并接收新账号购买申请。它不承担 Codex 使用、普通用户登录、公开注册、余额查询或账号自助管理。

公开页面
--------

1，根路径展示 Codex+++ 的核心价值、使用流程、功能说明、套餐、客户端下载和常见问题。
2，页面需要适配桌面和移动端，使用普通用户能理解的中文，不展示 API Key、endpoint、号池、模型路由或内部配置。
3，套餐只读取已启用、价格大于 0 且 Token 数量大于 0 的 Token 充值项。free 项和停用项不得成为官网购买选项。
4，客户端下载地址、版本和 SHA-256 复用后端部署配置。下载地址只允许 `http` 或 `https`。
5，官网必须明确说明 Codex+++ 为独立产品，与 OpenAI 不存在官方隶属关系。

购买申请
--------

1，访客提交套餐、联系方式、可选期望账号和可选备注。官网不得收集邮箱密码、OpenAI 密码、一次性验证码、支付密码或 Codex 授权材料。
2，提交时保存套餐名称、人民币价格和 Token 数量快照，后续修改 Token 充值项不得改变已提交订单。
3，购买者联系方式、期望账号和备注必须作为一个整体加密保存，不得出现在公开响应、审计详情或日志中。
4，公开接口按来源地址限流并使用蜜罐字段拦截简单机器人。创建成功只返回订单 ID、状态和人工联系说明。
5，购买申请状态为 `pending`、`contacted`、`fulfilled`、`rejected`。已完成或已拒绝订单不得再次处理。
6，首版采用人工联系和人工确认收款，不接自动支付。提交购买申请不表示付款成功。

后台处理
--------

1，已认证管理员可以查看解密后的购买信息、筛选状态、标记已联系、拒绝或完成订单。
2，完成订单前，管理员必须先创建或选择一个已存在且启用的普通用户账号。
3，完成订单时，后端将订单快照中的 Token 一次性写入该用户钱包，并形成来源为“官网购买：套餐名称”的 Token 流水。
4，完成和状态修改必须写入审计，但审计不得包含购买者联系方式和备注原文。
5，运营概览显示待处理官网购买申请数量，并能跳转到对应筛选结果。

验收标准
--------

1，访问 `/` 可以了解项目、下载客户端并提交购买申请；访问 `/admin` 仍进入管理后台。
2，公开套餐接口不返回 free 项、停用项或管理字段。
3，数据库或本地状态文件中找不到购买者信息明文。
4，不存在用户时无法完成订单；绑定有效用户后只入账一次。
5，官网和管理后台不使用浏览器原生 `prompt`、`alert` 或 `confirm` 完成订单操作。
