Token 充值项管理
===============

目标
----

让管理员创建和维护普通用户可申请的 Token 充值项。

参考项目
--------

本模块参考 Lago 的用量计量、预付 credits 和钱包思路，但 Codex+++ 首版只保留 Token 充值项和 token 余额所需的简单管理能力。

必须做到
--------

1，创建 Token 充值项。
2，启用和停用 Token 充值项。
3，配置 Token 充值项对应的 Codex+++ token 数量。
4，停用 Token 充值项后不得继续作为新充值选项。
5，配置 Token 充值项人民币展示价格。
6，配置 Token 充值项排序。
7，配置 Token 充值项简短说明。
8，记录 Token 充值项创建时间和更新时间。
9，Token 充值项列表支持搜索。
10，Token 充值项列表支持筛选。
11，Token 充值项列表搜索范围只搜索 Token 充值项名称。
12，Token 充值项列表筛选项只包含启用状态。
13，Token 充值项被停用后只影响新的充值请求，不影响历史充值记录。
14，价格大于 0 的 Token 充值项允许同一普通用户重复申请。
15，价格为 0 的 free Token 充值项不提供普通用户端申请动作。

不得混淆
--------

1，管理员管理 Token 充值项不是普通用户真实在线支付。
2，Token 充值项不是 Codex 账号余额。
3，Token 充值项确认后增加的是 Codex+++ token 余额。
4，Token 充值项价格不是真实自动支付通道。
5，Token 充值项价格允许为 0，价格为 0 时表示 free。
6，Token 充值项不表示模型选择、订阅周期或技术服务档位。

不得做到
--------

1，引入发票、税费、催收、优惠券或完整财务账务。
2，引入 Lago 运行时代码、数据库表名或权限模型。
3，把 Token 充值项管理做成普通用户 Web Token 充值项页。
4，把 Token 充值项做成不可重复充值的订阅式方案。
