模块拆分规则
============

定义
----

模块是可以独立描述、实现和验收的功能责任单元。

以下名称不是模块：

1，桌面客户端。
2，Web 管理后台。
3，后端服务。
4，共享契约。

这些名称只表示模块运行或被调用的位置。

拆分规则
--------

1，一个模块只描述一个主要责任。
2，一个模块必须能说明目标、必须做到和不得做到。
3，一个模块不得同时混写普通用户流程和管理员流程。
4，一个模块不得同时混写 Codex+++ token 余额和 Codex 账号余额。
5，一个模块不得把 API Key 自动切换写成普通用户手动配置。

术语边界
--------

1，Codex+++ 余额：首版以 token 表达，指普通用户提交充值请求并经管理员确认入账后，在 Codex+++ 内可消费的剩余 token。
2，Codex 账号余额：后端号池中某个上游 Codex 账号自身的可用余额或可用额度。
3，API Key 自动切换：Codex 账号余额不足、当前 API Key 不可用、当前 API Key 被停用或当前上游账号不可路由时，由后端 gateway 切换到可用 API Key 或等价可用路由。

禁止混淆
--------

1，Codex+++ token 余额不足不得触发 API Key 自动切换。
2，Codex 账号余额不足不得提示普通用户发起充值或补足 token。
3，普通用户不得查看、复制或手动选择 API Key。
