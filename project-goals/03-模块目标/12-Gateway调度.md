Gateway 调度
============

目标
----

承接 Codex 请求，并选择可用 API Key 完成上游调用。

参考项目
--------

本模块必须参考 sub2api 的 gateway 调度机制，重点参考鉴权、策略检查、API Key 状态检查、Codex 账号余额检查、可用账号或可用 API Key 选择、上游调用、失败切换和受控响应。

参考 sub2api 的结果必须落到 Codex+++ 自有 gateway，不得直接暴露 sub2api 路由、页面、权限模型或上游错误原文。

首版 gateway 不做任何旧接口兼容，不兼容 OpenAI、Anthropic、sub2api 或历史实现中的多接口形态。

Codex+++ gateway 只能暴露 Codex+++ 明确定义的唯一干净契约。不得保留根级 `/v1/chat/completions`、`/v1/responses`、`/v1/messages`、Images 或其他历史兼容路由作为 fallback。

允许保留唯一的 Codex 形态鉴权入口 `/api/codex/v1/models` 和 `/api/codex/v1/responses`。该入口属于后端受控适配能力，必须复用后端内部 gateway executor 的鉴权、余额检查、扣费、幂等、路由和审计逻辑，不得扩展成旧 OpenAI 兼容层。Windows 与 macOS 客户端在无本机 Codex 登录态，或已有登录账号通过本后端号池归属校验时，可以把该入口写入 Codex provider 配置；外部个人账号不得被覆盖。

必须做到
--------

1，验证普通用户身份和设备状态。
2，检查普通用户可用状态和 Codex+++ token 余额。
3，检查 API Key 状态。
4，检查 API Key 对应的 Codex 账号余额。
5，根据策略选择可用 API Key。
6，使用所选上游 Codex 账号调用 Codex 官方 Responses 端点。
7，返回受控响应。
8，发现旧路由、旧字段、旧配置或旧兼容层时必须清理。
9，普通用户客户端只把 Codex 请求送入 `/api/codex/v1`，不得参与上游账号选择。
10，同一 Codex 会话的后续请求必须持久保留上游账号粘性；正常重启或更新部署不得无故换号。绑定标识应使用不可逆摘要，并设置过期时间。
11，Responses SSE 工具事件和 `x-codex-turn-state` 必须返回本机 Codex；本机工具由本机 Codex 执行。
12，成功用量记录必须保存可用的 Codex 会话标识，供管理员按账号和时间段统计任务量、Token 总量和单任务平均 Token；该内部标识不得暴露给普通用户。
13，Gateway 必须兼容 Codex Responses 请求的 `Content-Encoding: zstd`。压缩内容必须在 JSON 校验和字段清理前解压，并分别限制压缩前后的请求体大小；不得把客户端压缩头与压缩二进制直接转发给上游。
14，Gateway 的预扣、实际扣费、用量记录、账本、幂等完成和审计必须在同一个 PostgreSQL 事务中提交；中断或失败不得留下永久预扣。
15，Responses SSE 必须边接收边转发并及时刷新；不得等待完整响应后再一次性返回。最终 usage 到达后再完成原子结算。
16，每个上游账号必须有原子并发租约。部署 Redis 时，租约、续期、释放和每日检查锁必须跨进程生效；Redis 不可用时新请求失败关闭。
17，同一普通用户在运行租约有效期间必须优先保持当前健康上游账号，不得因新建 Codex 会话或开始新任务而轮换账号。只有当前上游额度耗尽、认证失效、限流、路由不可用或服务不可用时才允许受控切换。
18，每个上游账号必须限制同时绑定的普通用户数量，默认上限为 2，并允许通过部署配置调整。上限按不同普通用户去重，不按同一用户的请求数或设备数重复计算；账号行必须显示“正在使用人数/上限”。
19，客户端运行心跳必须续期普通用户的稳定上游绑定；用户全部设备停止或运行租约到期时释放该用户占用。未完成首个任务的运行客户端只能显示为待分配，不得虚构号池账号归属。

不得混淆
--------

1，Codex+++ token 余额不足时不得继续调度 API Key。
2，Codex 账号余额不足时可以切换 API Key。
3，上游错误不得原文透传给普通用户。
4，不得把 gateway 写成 OpenAI、Anthropic、sub2api 或旧实现兼容层。
5，不得在后端为普通用户任务创建 app-server thread 或 turn，不得在服务器目录执行用户项目工具。
6，不得以轮询、随机选择或每任务最少使用策略覆盖仍健康的用户级稳定绑定。
