普通用户客户端界面
==================

目标
----

让普通用户默认看到足够简单、可直接使用的客户端界面，同时在需要时进入高级模式查看记录、状态和脱敏诊断。

参考项目
--------

本模块参考 Codex++ 的入口分层：静默启动入口负责直接启动，管理工具面板承载检查、修复、更新、配置和增强管理。

Codex+++ 不复制旧 Codex++ 高级工作台。Codex+++ 的默认界面必须是普通用户简洁模式，高级模式只承载普通用户可理解的记录、状态和脱敏诊断。

必须做到
--------

1，普通用户登录成功后默认进入简洁模式。
2，简洁模式显示账号、最近充值状态、剩余 token 精确数字、最近 7 日 token 用量迷你趋势图、服务状态、启动状态和一个明确主动作。
3，简洁模式提供申请 Token 充值项入口，但不得把充值做成普通用户浏览器自助页面或外部交易页面。
4，简洁模式提供进入高级模式的按钮。
5，高级模式显示 token 用量明细、token 收入和消耗流水、历史充值请求、设备状态、安全状态和脱敏诊断。
6，高级模式中的 token 用量明细需要分页。
7，token 用量明细每页默认显示 10 条。
8，历史充值请求放在高级模式中展示。
9，历史充值请求每页默认显示 10 条。
10，高级模式提供返回简洁模式的动作。
11，启动状态至少区分未启动、正在启动、已启动和启动失败。
12，启动失败是普通用户本机行为，客户端只在本机显示中文提示和下一步动作。
13，管理员后台不得做成监控每个普通用户本机启动过程的面板。
14，余额不足时客户端不额外显示“余额不足”提示；后端不再提供后续 Codex 账号、API Key 或等价可用路由。
15，高级模式的 token 收入和消耗流水采用最近 30 日正负柱状图加流水表。
16，简洁模式中的充值入口登录后常驻显示，不仅在 token 不足时出现。
17，Token 充值项区域展示所有已启用 Token 充值项；价格大于 0 的 Token 充值项显示“申请充值”，价格为 0 的 free Token 充值项不显示申请动作。
18，普通用户高级模式脱敏诊断首版只显示本机 Codex 是否检测到、配置是否写入成功、最近一次启动失败原因。

不得展示
--------

1，账户凭据。
2，API Key 原文。
3，proxy。
4，endpoint。
5，base_url。
6，内部 gateway 配置。
7，上游错误原文。
8，普通用户可编辑的 provider、模型选择或技术服务档位。
9，旧 Codex++ 高级工作台入口。
10，诊断原文、路径细节或内部配置明细。
