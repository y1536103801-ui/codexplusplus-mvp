Codex+++ 项目期望与目标
=======================

用途
----

本目录记录我对 Codex+++ 当前期望和目标的理解。

它用于回答三个问题：

1，Codex+++ 想成为一个什么产品。
2，Codex+++ 当前必须完成哪些目标。
3，Codex+++ 当前明确排除哪些方向。

阅读顺序
--------

1，`01-总体期望.md`
2，`02-产品目标.md`
3，`03-模块目标.md`
4，`04-边界与非目标.md`
5，`05-实现补充要求.md`

表达规则
--------

1，`Codex+++` 指本目录描述的产品。
2，`Codex++` 指必须参考并可复用本机能力的上游客户端项目。
3，`sub2api` 指必须参考“导入 Codex 上游账户形成号池、把号池资源转成可调度 API Key、按 Codex 账号余额和 API Key 状态切换可用 API Key”的核心机制，以及账号分组、额度、用量和风控机制的项目。
4，`普通用户` 指只使用桌面客户端的人。
5，`管理员` 指只使用 Web 管理后台的人。
6，`Codex+++ 余额` 首版以 token 表达，指普通用户提交充值请求并经管理员确认入账后，在 Codex+++ 内可消费的剩余 token。
7，`Codex 账号余额` 指后端号池中某个上游 Codex 账号自身的可用余额或可用额度。

外部参考项目要求
----------------

Codex++ 和 sub2api 不是背景材料，也不是可有可无的名称引用。它们是 Codex+++ 当前目标文档和后续实现必须对照的两个外部参考项目。

当前默认参考来源如下：

1，Codex++：`https://github.com/BigPizzaV3/CodexPlusPlus`
2，sub2api：`https://github.com/Wei-Shaw/sub2api`

如果后续指定 fork、固定提交或本地路径，则以指定来源为准；未指定前，后续实现必须先对照上述公开仓库。

Codex++ 必须作为桌面客户端和本机能力参考。目标文档中涉及 Tauri 桌面壳、本机 Codex 检测、受控配置清理、启动流程、更新和脱敏诊断时，都必须明确落到 Codex++ 对应能力。

sub2api 必须作为后端号池、API Key、gateway、额度、用量和风控机制参考。目标文档中涉及导入 Codex 上游账户形成号池、把号池资源转成可调度 API Key、按 Codex 账号余额和 API Key 状态切换可用 API Key、账号分组、额度、用量、风控和审计时，都必须明确落到 sub2api 对应机制。

参考这两个项目不表示复制它们的用户界面、运行时代码、数据库表名、权限模型或历史产品形态。Codex+++ 必须形成自己的桌面客户端、Web 管理后台、后端服务和共享契约，但实现目标不得脱离上述两个项目提供的参考依据。

本目录只描述期望和目标，不描述历史实现细节。
