边界与非目标
============

项目边界
--------

Codex+++ 当前由桌面客户端、Web 管理后台、后端服务和共享契约组成。

它不是单纯的前端换皮。

它不是 sub2api 改造项目。

它不是普通用户 Web 自助平台。

它也不是把旧 Codex++ 高级工作台包装后交给普通用户使用。

它不是 OpenAI、Anthropic、sub2api 或历史实现的兼容网关。

角色边界
--------

普通用户只使用桌面客户端。

管理员只使用 Web 管理后台。

后端服务只向桌面客户端、Web 管理后台和 gateway 调用方提供受控 API。

普通用户不直接接触 Admin API、号池、账户凭据、内部 gateway 配置或系统诊断原文。API Key 只能由后端 gateway 受控生成、使用和切换，不得进入普通用户可查看、复制或手动配置的客户端流程。

首版实现只覆盖 Windows 桌面客户端。macOS、Linux、移动端和浏览器插件不作为首版完成条件。

当前非目标
----------

当前 MVP 不包含：

1，普通用户 Web 页面。
2，普通用户注册。
3，管理员公开注册。
4，邀请码注册。
5，OAuth 注册。
6，普通用户 Web 选择 Token 充值项。
7，普通用户 Web 申请 Token 充值项。
8，普通用户 Web 升级、续费或取消 Token 充值项。
9，独立于客户端的 Web 支付。
10，独立于客户端的充值中心。
11，独立于桌面客户端充值流程的交易记录页面。
12，独立于桌面客户端充值流程的财务明细页面。
13，兑换。
14，普通用户浏览器账户页面。
15，普通用户账户凭据或 API Key 手动配置。
16，普通用户可达支持工作台。
17，脚本市场。
18，推荐内容。
19，多租户白标。
20，复杂分销。
21，完整财务账单。
22，首版真实自动支付通道。
23，普通用户自助充值。
24，macOS 客户端。
25，Linux 客户端。
26，OpenAI、Anthropic、sub2api 或历史实现的接口兼容层。
27，旧路由、旧字段、旧页面、旧配置或旧 fallback。

措辞边界
--------

“Token 充值项”在 Codex+++ 当前目标中指管理员创建并维护、普通用户可在客户端选择并提交充值请求的 token 余额补给模板。

“Codex+++ 余额”在首版中以 token 表达，指普通用户提交充值请求并经管理员确认入账后，在 Codex+++ 内可消费的剩余 token。Codex+++ token 余额不足时，客户端不主动触发充值、联系管理员或 API Key 切换；后端不再提供后续 Codex 账号、API Key 或等价可用路由，由 Codex 自身呈现不能继续使用的提示。

“Codex 账号余额”指后端号池中某个上游 Codex 账号自身的可用余额或可用额度。Codex 账号余额不足时，后端可以切换到其他可用 API Key 或等价可用路由。

“充值请求”若没有特别说明，默认表示普通用户登录后在桌面客户端内选择 Token 充值项并提交申请，由管理员在 Web 管理后台确认入账；首版不接真实自动支付通道，不要求普通用户填写备注、金额、token 或原因。

“Token 充值项价格”指普通用户客户端和管理员后台展示的人民币价格。Token 充值项价格首版只用于展示和充值确认，不表示接入真实自动支付通道、发票、税费或完整财务账务。Token 充值项价格允许为 0，价格为 0 时表示 free；free Token 充值项不提供普通用户端申请动作。

“高级模式”若出现在当前目标中，只能表示普通用户登录后的高级模式面板，不表示旧 Codex++ 高级工作台。

首版 Token 充值项只提供 Codex 账号服务对应的 token，不提供模型选择、订阅周期、技术服务档位或普通用户需要理解的策略配置。

“注册”若出现在当前目标中，默认表示公开创建账号；当前 MVP 不做。设备登记可以作为内部技术动作存在，但不得在普通用户界面表达为注册。

“参考 Codex++”只表示参考和复用适合桌面客户端的一键使用能力，包括 Tauri 桌面壳、本机 Codex 检测、受控配置清理、启动流程、更新和脱敏诊断；不表示复用旧 Codex++ 高级工作台、普通用户不可理解的配置界面或历史产品入口。

“参考 sub2api”只表示参考导入 Codex 上游账户形成号池、把号池资源转成可调度 API Key、按 Codex 账号余额和 API Key 状态切换可用 API Key、额度、用量和风控等机制；不表示复用 sub2api 运行时代码、路由、页面、数据库表名、权限模型或前端资产。

“干净”表示只保留 Codex+++ 当前唯一目标所需的内容。发现旧内容、旧接口、旧页面、旧配置或旧兼容层时，应立即清理，不得保留为兼容 fallback。

完成状态
--------

我理解的完成状态是：

1，普通用户可以在 Windows 桌面客户端完成登录、选择 Token 充值项并提交充值请求、查看 Codex+++ token 余额、自动准备环境并启动 Codex。
2，管理员可以在 Web 管理后台完成初始化、登录、首轮更改账户密码、运营概览查看、用户管理、Token 充值项管理、充值确认、token 调整、号池管理、设备管理、用量查看和审计查看。
3，后端可以从 Codex 账户号池生成或维护可调度 API Key，并在 gateway 中完成鉴权、策略检查、Codex+++ token 余额检查、Codex 账号余额检查、API Key 选择、上游调用、扣费、幂等和审计。
4，后端 gateway 在 Codex 账号余额不足或当前 API Key 不可用时切换可用 API Key 或等价可用路由；普通用户不需要理解、查看、复制或手动配置 API Key。
5，客户端在 Codex+++ token 余额不足时不主动触发充值、联系管理员或 API Key 切换；后端不再提供后续 Codex 账号、API Key 或等价可用路由，不得把 Codex+++ token 余额不足处理成 API Key 自动切换。
6，当前产品中不存在普通用户 Web 页面、注册、普通用户浏览器自助页面、外部交易页面或真实自动支付通道。
7，首版完成状态只按 Windows 客户端验收。
8，gateway 只存在 Codex+++ 唯一干净契约，不存在 OpenAI、Anthropic、sub2api 或历史实现兼容层。
