总体期望
========

核心期望
--------

Codex+++ 的目标是成为 Codex++ 的面向普通用户的升级版。

这里的“升级版”不是简单改名，也不是把旧界面继续堆功能，而是把 Codex 的安装、配置、模型、账号、额度和启动过程封装成普通用户能理解的使用体验。

普通用户不需要理解技术细节。普通用户只需要知道：

1，如何登录。
2，如何在客户端发起充值请求。
3，当前服务是否可用。
4，当前 Codex+++ token 余额和最近充值状态是什么。
5，能否启动 Codex。
6，Codex 账号余额不足或当前 API Key 不可用时，后端 gateway 是否已经切换到可用 API Key 或等价可用路由。
7，异常时是否需要联系管理员。

尚未购买的访客可以通过官网了解项目、下载 Windows 或 macOS 客户端、查看当前可售 Token 套餐并提交账号购买申请。官网不提供普通用户登录、公开注册、自动支付、余额查询或账号自助管理。

参考来源期望
------------

Codex+++ 当前目标必须明确参考两个开源项目：Codex++ 和 sub2api。

Codex++ 是桌面客户端和本机能力参考。Codex+++ 需要参考 Codex++ 中适合普通用户一键使用的能力，包括 Tauri 桌面壳、本机 Codex 检测、受控配置清理、启动流程、更新和脱敏诊断。

sub2api 是后端调度机制参考。Codex+++ 需要参考 sub2api 中导入 Codex 上游账户形成号池、把号池资源转成可调度 API Key、按 Codex 账号余额和 API Key 状态切换可用 API Key、账号分组、额度、用量、风控和审计的机制。

参考 Codex++ 和 sub2api 是实现目标的依据，不是复制它们的旧界面、普通用户 Web 自助业务、运行时代码或历史项目形态。Codex+++ 自有官网只承担介绍、客户端下载和人工审核的账号购买申请。

管理员需要掌握系统配置和资源分配。管理员负责：

1，初始化系统。
2，登录 Web 管理后台。
3，创建和管理普通用户。
4，管理 Token 充值项。
5，管理 Codex 账户号池、账户凭据和后端生成的 API Key。
6，确认普通用户的充值请求，并为普通用户充值 Codex+++ token。
7，管理普通用户的 Codex+++ token 余额、最近充值状态和可用状态。
8，查看设备、用量、token 流水和审计。
9，查看管理员运营概览，包括待确认充值、可用号池账号和今日 token 用量。
10，处理官网账号购买申请，联系购买者，并在确认收款后创建或绑定普通用户账号完成初始 Token 入账。

项目质量期望
------------

Codex+++ 的目标是保持干净的项目边界。

干净的含义是：

1，不把 sub2api 当作 Codex+++ 后端继续改造。
2，不把 sub2api 的普通用户浏览器账户中心、自助业务或公开注册流程迁入 Codex+++。
3，不把旧 Codex++ 高级工作台当作普通用户界面。
4，不让历史截图、历史页面或临时实现反向定义当前目标。
5，不把客户端充值请求与官网账号购买申请混为一条记录：前者属于已登录用户充值，后者属于访客购买新账号，二者分别审核和入账。
6，桌面客户端实现范围覆盖 Windows 与 macOS；两种客户端必须遵守同一后端契约、账号边界、计费规则和一键使用流程。Linux 和移动端不作为当前完成条件。

用户体验期望
------------

Codex+++ 的普通用户体验目标是足够简单。

普通用户打开桌面客户端并登录成功后，默认看到简洁模式。简洁模式只表达账号、最近充值状态、剩余 token、最近 7 日 token 用量迷你趋势图、服务状态、启动状态和一个明确主动作。

普通用户未登录时，不得看到高级模式、注册、充值、支付、Token 充值项选择、账户凭据、API Key、proxy、endpoint、base_url、诊断、安装维护或外部网页入口。

普通用户已登录后，可以在客户端选择 Token 充值项并提交充值请求、查看剩余 token、点击打开 Codex 并直接正常使用。充值入口登录后常驻显示，不仅在 token 不足时出现。启动 Codex 时，如果本机没有 Codex 登录态，客户端向后端获取当前普通用户专属的 Codex+++ 网关访问 key，写入 Codex `auth.json` 的 `auth_mode = "apikey"` 和 `OPENAI_API_KEY`，并写入 Codex+++ provider 配置。用户网关访问 key 与号池内部路由 API Key 必须分离，不绑定上游账号；后端收到请求后才动态选择号池路由。Codex 账号余额不足或当前内部路由不可用时，后端 gateway 应切换到可用 API Key 或等价可用路由。Codex+++ token 余额不足时，客户端不主动触发充值、联系管理员或 API Key 切换；后端不再提供后续可用路由，由 Codex 自身呈现不能继续使用的提示。

普通用户已登录后，可以点击高级模式按钮进入高级模式面板。高级模式只用于展示 token 用量明细、token 收入和消耗流水、历史充值请求、设备、安全状态和脱敏诊断，不得变成旧 Codex++ 高级工作台。脱敏诊断首版只显示本机 Codex 是否检测到、本机 Codex 是否正在运行、本机 Codex 版本和最近一次启动失败原因。
