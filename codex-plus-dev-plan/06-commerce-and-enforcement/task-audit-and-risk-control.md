# 审计与风控

## 目标

记录 Codex++ 用户关键操作和异常行为，为售后、封禁、限流和问题排查提供依据。

## 涉及项目

- sub2api-main：ops/user error/audit 相关服务
- sub2api-main：gateway usage/error logging
- CodexPlusPlus-main：诊断日志导出后续对接

## 输入契约

- 用户 ID。
- API Key ID。
- 设备 ID。
- gateway policy 错误码。
- 请求 ID、配置版本、模型 ID、风险标签、计量事件 ID。

## 输出行为

记录：

- 登录/设备登记。
- bootstrap 调用。
- Key 创建/轮换。
- 模型拒绝、余额不足、速率超限。
- 管理员撤销设备或封禁用户。
- 配置发布、回滚、权益修正、支付补偿和用量回补。
- 高风险行为：异常设备切换、共享账号特征、短时高频失败、模型探测。

## 解耦要求

- 审计记录不存完整 Key。
- 风控策略由后台配置或管理员操作触发，不由客户端决定。
- 风控结果可以影响 bootstrap 状态和网关拒绝，但客户端不得自行判定封禁。

## 禁止改动范围

- 不实现自动封号策略，除非已有配置明确要求。
- 不改支付签名日志。
- 不收集用户项目代码内容。
- 不记录 prompt、代码文件内容或用户本地路径等敏感业务内容。

## 测试要求

- 敏感字段脱敏。
- 审计记录按用户和设备可查询。
- 网关拒绝事件可被管理员定位。
- 可按请求 ID 串联 bootstrap、网关拒绝、用量事件和管理员操作。
- 风控事件可被售后查询，但默认不暴露给普通用户。

## 交付物

- 审计事件定义。
- 日志记录点。
- 查询说明和测试。
- 风控标签、脱敏规则和事件保留策略。
