## User-Facing Scope

- Codex++ Cloud is documented as the managed provider created by the backend after the user completes the browser login and desktop binding flow.
- The client copy uses backend-configured plan, model and quota language. User-visible values are treated as a configuration snapshot returned by the service, not hardcoded local entitlement rules.
- Old manual provider preservation is stated as a compatibility boundary: user-managed providers remain user-owned and are not overwritten by managed cloud provider setup.
- Failure states are covered in user language and ordered for support triage: not purchased, expired, insufficient balance, device revoked, model unavailable and local configuration failure.

## Release-Candidate User Copy Decision

This evidence set treats the user guide as final for product-copy hygiene. It does not claim that browser handoff, payment, desktop launch or gateway request execution has completed; those proofs stay in their dedicated release evidence lanes.
