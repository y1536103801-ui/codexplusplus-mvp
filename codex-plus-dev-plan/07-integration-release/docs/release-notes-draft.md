# Codex++ 07 Integration Release 发布说明

Status: final for docs/product-copy evidence
Release evidence boundary: E2E, package, compatibility and business readiness remain separate lanes.

## 发布范围

本次发布阶段验证 Phase 1 本地/测试 MVP 闭环：后台开通或测试支付、用户登录、设备刷新、bootstrap、写入 `Codex++ Cloud`、启动 Codex、完成一次请求并查到 usage/log。

本说明不代表生产可售卖发布。生产上线仍需通过公网部署、HTTPS、生产密钥管理、真实或沙盒支付回调、监控告警、备份恢复、安全合规和生产 go/no-go。

## 用户可见变化

- 登录后自动拉取当前账号的托管配置快照。
- 自动写入或更新 `Codex++ Cloud` 托管供应商。
- 首页展示服务可用、未购买、套餐过期、余额不足、设备撤销、模型不可用、网关不可用和本地配置失败等状态。
- 普通用户默认不需要填写 API Key、Base URL 或模型协议。
- 旧手动供应商配置保留在高级入口中。

## 管理员可见变化

- 套餐、价格、模型、额度、功能开关和续费文案由后台配置版本驱动。
- 配置发布应记录版本、操作者、灰度范围、发布时间和回滚点。
- 权益、设备、用量、网关拒绝和人工修正进入审计与对账链路。

## 配置与契约

- 配置版本：待 Module I/J 汇总实际证据后填写。
- 契约版本：待 Module I/J 汇总实际证据后填写。
- Snapshot 行为：客户端消费最小运行快照，不接收完整后台运营规则。
- 可变销售内容：以后台配置中心当前发布版本为准，客户端不内置固定价格、模型或额度。

## 回滚路径

- 配置回滚：管理员在 Config Registry 回滚到上一稳定版本，客户端刷新 bootstrap 后体现。
- 后端回滚：保持 `bootstrap`、`usage`、`devices`、`redeem` 契约兼容，避免旧客户端遇到未知破坏性字段。
- 桌面端回滚：安装包回滚不得删除旧手动供应商，不得暴露完整 Key 或令牌。
- 权益修正：通过账本补偿和审计记录处理，不直接覆盖不可追踪数据。

## 兼容影响

- 旧用户手动供应商配置必须保留。
- 直接密码登录仅作为兼容路径；生产默认路径为浏览器授权。
- 高级供应商配置默认隐藏，是否显示由后台功能开关控制。

## 已知风险

- 最终 E2E 证据仍需由 R1/R2/R3 和 Module I/J 汇总。
- 安装包证据、兼容迁移证据、配置回滚证据和用量对账证据尚未附在本 R4 文档产物中。
- 生产上线仍需独立 QA/staging、安全、隐私合规、SLO、成本滥用和客服运营门禁。
