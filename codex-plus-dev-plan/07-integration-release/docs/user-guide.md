# Codex++ 用户指南

Status: final for docs/product-copy evidence
Audience: end users and support
Release evidence boundary: E2E, package, compatibility and business readiness remain separate lanes.

## 核心口径

用户不需要理解 API、Base URL、模型协议或供应商 Key。购买或开通权益后，登录 Codex++ Manager，客户端会拉取后台配置快照并自动写入 `Codex++ Cloud` 托管供应商。

实际套餐名、价格、可用模型、额度、续费入口和状态文案以后台配置中心当前版本为准，客户端不承诺内置固定销售表。

## 首次使用流程

1. 获得权益
   - 通过当前销售入口购买、兑换码兑换，或由管理员开通测试权益。
   - 如果显示未开通，先确认登录账号是否与购买账号一致。

2. 安装 Codex++ Manager
   - 使用当前发布包安装。
   - 首次打开会检测 Codex App、配置目录、网络和本机权限。
   - 按客户端提示完成 Codex 安装或权限修复。

3. 浏览器授权登录
   - 点击登录后在浏览器完成授权。
   - 确认桌面端显示的 6 位确认码与网页一致。
   - 不向任何人提供令牌、支付信息或 API Key。

4. 自动配置
   - 客户端拉取 `GET /api/v1/client/bootstrap` 快照。
   - 自动写入或更新 `Codex++ Cloud`。
   - 旧手动供应商保留在高级配置中。

5. 启动 Codex
   - 首页状态为可用时点击启动。
   - 启动失败时先查看客户端给出的状态原因和修复入口。

## 第一条 Codex 任务

新用户可按下面步骤完成第一条任务：

1. 在 Codex 中选择要处理的项目目录。
2. 粘贴任务模板：`帮我阅读这个项目，说明启动方式，并找出首页按钮点击无反应的可能原因。`
3. 阅读 Codex 的计划和修改范围。
4. 执行前确认不会提交密码、支付信息、真实密钥或隐私数据。

## 常见状态说明

| 状态 | 说明 | 推荐动作 |
| --- | --- | --- |
| 服务可用 | 当前账号命中的后台配置允许使用 | 启动 Codex |
| 未购买 | 后台权益不存在或未生效 | 购买、兑换或联系管理员 |
| 套餐过期 | 权益有效期已结束 | 按当前续费入口恢复 |
| 余额不足 | 额度不足以继续请求 | 充值、续费或等待配置规定的刷新 |
| 设备撤销 | 当前设备已被后台撤销 | 重新绑定或联系管理员 |
| 模型不可用 | 模型未开放、下架或被网关拒绝 | 选择可用模型或等待后台配置更新 |
| 网关不可用 | 服务端或网络异常 | 稍后重试并导出诊断信息 |
| 本地配置失败 | 托管供应商写入失败 | 使用修复入口并检查本机权限 |

## 支持与隐私提醒

- 诊断日志应脱敏，不展示完整 API Key、JWT、Authorization header 或支付凭证。
- 客服定位时优先使用账号、设备、请求 ID、配置版本和错误码。
- 用户侧不需要知道真实上游供应商凭证或内部风控规则。
