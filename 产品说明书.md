# Codex++ 产品说明书

## 重要说明

Codex++ 客户端不内置固定价格、固定模型、固定额度或固定套餐权益。用户看到的套餐、模型、额度、续费入口、状态文案和功能开关来自后台配置中心发布的当前配置版本。本文档中的名称均为说明性口径，实际销售内容以后台配置和发布说明为准。

## 用户指南

### 1. 购买或获得权益

- 新用户可以通过当前销售入口、兑换码或管理员开通获得权益。
- 支付、兑换或人工开通完成后，权益进入账号。
- 如果客户端显示“未购买”或“未开通”，请使用页面提供的购买/兑换入口，或联系管理员确认账号权益。

### 2. 安装 Codex++ Manager

- 下载当前发布包并完成安装。
- 首次打开后，Manager 会检测 Codex App、配置目录、网络连通性和本机权限。
- 如果 Codex 未安装，客户端应展示下载和安装辅助。
- 如果本机权限或配置目录不可写，客户端应展示一键修复、路径说明或日志导出入口。

### 3. 登录账号

- 推荐生产路径为浏览器授权登录。
- 桌面端会显示授权链接和 6 位确认码。
- 不要向任何人提供访问令牌、刷新令牌、支付信息或上游 API Key。

### 4. 自动配置 `Codex++ Cloud`

- 登录成功后，客户端会调用 bootstrap 接口拉取当前用户可用快照。
- 客户端自动写入或更新 `Codex++ Cloud` 托管供应商。
- 普通用户无需填写 Base URL、API Key、wire API 或模型协议。
- 旧手动供应商会保留在高级配置里，不会因登录、刷新或退出而被删除。

### 5. 启动 Codex

- 首页显示服务状态、设备状态、托管供应商状态和启动按钮。
- 状态为可用时，点击“启动 Codex”即可进入 Codex。
- 第一次使用可以复制这个任务模板：`帮我阅读这个项目，说明启动方式，并找出首页按钮点击无反应的可能原因。`
- 开始任务前，先选择项目目录，再描述希望 Codex 完成的目标。

## 常见状态

| 状态 | 含义 | 用户动作 |
| --- | --- | --- |
| 服务可用 | 当前账号、设备、模型和额度均可使用 | 点击启动 Codex |
| 未购买 | 当前账号没有可用权益 | 按当前入口购买、兑换或联系管理员 |
| 套餐过期 | 权益有效期已结束 | 续费后刷新客户端 |
| 余额不足 | 当前额度不足以继续请求 | 充值、续费或等待后台配置的额度刷新 |
| 设备撤销 | 当前设备不再允许使用 | 重新绑定或联系管理员 |
| 模型不可用 | 模型下架、未开放或被网关拒绝 | 切换到后台配置允许的模型 |
| 网关不可用 | 服务端或网络暂不可达 | 稍后重试，必要时导出诊断信息 |
| 本地配置失败 | `Codex++ Cloud` 写入失败 | 使用一键修复或检查本机权限 |

## 管理员指南

### 配置套餐、价格和额度

- 在 Admin Console 或配置中心维护套餐、价格、有效期、额度、模型组、设备策略和续费文案。
- 每次变更通过 Config Registry 形成配置版本。
- 客户端只展示当前 bootstrap 快照，不应依赖本地固定销售表。

### 配置模型和默认模型

- 管理员维护展示模型、真实路由、模型组权限、上下架状态和回退策略。
- 默认模型来自当前用户命中的配置版本。
- 模型不可用时，Data Plane 必须在请求路径拒绝，客户端展示可理解原因和下一步动作。

### 配置功能开关

- 高级供应商配置、Codex 增强、安装辅助、续费入口和诊断入口都应由功能开关控制。
- 灰度时按用户、设备、套餐或环境范围发布，不要求客户端发版。

### 权益、设备和审计

- 管理员可以查看用户权益摘要、设备列表、Key 摘要、用量、拒绝原因和审计记录。
- 设备撤销后，bootstrap 或启动路径应给出 `device_revoked` 一类明确状态。
- 审计记录应覆盖管理员操作、配置变更、权益调整、风险处置和人工修正。

## 运营与发布

### 灰度发布

- 新套餐、模型、额度和功能开关先进入草稿并通过校验。
- 发布时记录配置版本、灰度范围、操作者、发布时间和回滚点。
- 灰度期间对比 bootstrap 成功率、网关拒绝率、用量事件和客服反馈。

### 回滚

- 配置错误：回滚到上一稳定配置版本，客户端刷新快照后恢复。
- 后端错误：回滚服务端版本时保持契约兼容，不让客户端遇到未知字段缺失或未知状态。
- 桌面端错误：安装包回滚不能删除旧手动供应商，必要时保留手动恢复说明。
- 权益错误：通过账本修正和审计记录补偿，不直接改写不可追踪数据。

### 用量对账

- 请求可预扣，请求结束后按真实消耗结算。
- 失败请求需要回补或标记为不计费。
- 用量事件必须能关联用户、设备、请求 ID、模型、配置版本和拒绝原因。
- 支付、退款、补偿和人工修正需要进入对账和审计流程。

## 发布前检查

- E2E 已覆盖购买或测试开通、登录、设备刷新、bootstrap、写 provider、启动 Codex、一次请求成功和 usage/log 可查。
- 失败路径已覆盖未购买、套餐过期、余额不足、设备撤销、模型不可用、网关不可用和本地配置失败。
- 文档和 HTML 展示页未承诺固定价格、固定模型或固定额度。
- 发布说明包含配置版本、契约版本、兼容影响、回滚路径和已知风险。
- 未通过生产 go/no-go 前，Phase 1 MVP 只能作为本地/测试验收闭环，不作为正式可售卖生产发布。
