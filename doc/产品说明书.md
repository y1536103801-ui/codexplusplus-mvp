\# Codex++ 最终成品说明书



\## 产品定位

Codex++ 是一款面向开发者的“一体化 Codex 增强客户端”。用户购买套餐后，只需登录 Codex++，即可自动获得可用服务配置，并直接启动 Codex 使用，无需理解或手动填写 `Base URL`、`API Key`、模型协议等复杂参数。



产品由两部分组成：



\- `Codex++ 客户端`：基于现有 CodexPlusPlus，负责安装、登录、自动配置、启动 Codex、增强注入、状态诊断和自动修复。

\- `Codex++ 云服务后台`：基于 Sub2API，负责用户、套餐、支付、余额、API Key、账号池、请求转发、计费、限流和管理后台。



\## 用户侧成品形态

用户安装后看到的是一个简化版 Codex++ Manager：



\- 首页显示服务状态、套餐余额、今日用量、当前可用模型和“一键启动 Codex”。

\- 首次使用时引导用户登录账号。

\- 登录后自动拉取专属服务配置并写入本机 Codex 配置。

\- 用户不需要看到 API Key，也不需要填写供应商信息。

\- 余额不足、套餐过期、服务异常时，客户端显示明确原因和续费入口。

\- 高级用户仍可进入“高级供应商配置”，手动维护自有 API 或第三方中转。



推荐主流程：



1\. 用户购买套餐。

2\. 下载并安装 Codex++。

3\. 打开 Codex++，登录购买账号。

4\. 客户端自动检测套餐和余额。

5\. 客户端自动配置 Codex 服务。

6\. 用户点击“一键启动 Codex”开始使用。

\## 核心接口

客户端启动后调用云端引导接口：



`GET /api/v1/client/bootstrap`



返回内容包括：



\- 网关地址

\- 用户专属 API Key

\- 默认模型

\- 协议类型

\- 当前套餐

\- 剩余额度

\- 是否可用

\- 公告和版本策略

\- 功能开关



设备绑定接口：



`POST /api/v1/client/devices`



用于记录设备、平台、客户端版本、Codex 版本和最后活跃时间。



用量查询接口：



`GET /api/v1/client/usage`



用于客户端展示余额、套餐、今日用量、限流状态和续费入口。



\## 商业闭环

用户购买路径：



\- 官网或后台购买套餐

\- 支付成功后 Sub2API 自动为用户充值或开通套餐

\- 用户登录 Codex++ 后自动获得服务配置

\- 使用过程中按 Token 计费或套餐扣量

\- 余额不足时客户端提示续费

\- 续费成功后自动恢复服务



默认销售形态：



\- 个人月卡

\- 个人按量包

\- 高级模型包

\- 团队套餐

\- 企业定制套餐



辅助能力：



\- 兑换码用于分销、补偿和活动发放

\- 管理员可手动调整余额

\- 可通过后台限制单用户并发、每日额度和模型权限



\## 成品验收标准

\- 新用户购买后，登录 Codex++ 即可启动 Codex 并完成一次可用请求。

\- 用户全程无需手动填写 `Base URL`、`API Key` 或模型协议。

\- 余额不足、套餐过期、服务被禁用时，客户端能准确提示原因。

\- 支付成功后，用户额度能自动到账并恢复使用。

\- 管理员能在后台看到用户、订单、余额、API Key、设备、用量和异常状态。

\- 现有 Codex++ 增强能力继续可用，包括插件入口增强、会话删除、Markdown 导出、Timeline、用户脚本等。

\- 高级供应商配置不删除，但默认隐藏，避免普通用户误操作。



\## 默认决策

\- 保留登录体系，作为购买、套餐、设备和售后绑定方式。

\- 不在安装包中内置共享 Key，避免泄露和滥用。

\- 客户端只保存 Sub2API 用户侧 Key，不暴露上游真实账号凭证。

\- Codex++ 默认进入托管模式，手动 API 配置作为高级模式。

\- 第一版优先实现“买完即用”的完整闭环，不急于重构全部供应商配置系统。
