\# Codex++ 买完即用产品方案书



\## 摘要

目标是把 `CodexPlusPlus-main` 的桌面端增强启动器，和 `sub2api-main` 的 API 网关、计费、支付、账号池能力组合成一款“购买后直接使用”的 Codex++ 产品。



核心体验：用户下载安装 Codex++，登录购买账号，客户端自动领取专属服务凭证并完成 Codex 配置，用户不再手动填写 `Base URL`、`API Key`、协议、模型等复杂项。



产品主张：不用找 API、不用配置 Key、不用理解中转，买完打开就能跑 Codex。



\## 关键表现内容

\- 首页卖点：买完即用、稳定中转、自动配置、余额透明、Codex 原生体验增强。

\- 新手路径：下载安装 Codex++ -> 登录账号 -> 检测套餐/余额 -> 自动初始化服务 -> 一键启动 Codex。

\- 客户端首页：显示“服务可用状态、套餐余额、今日用量、当前模型、启动按钮、修复按钮”，隐藏复杂供应商配置。

\- 高级设置：保留现有供应商配置能力，但默认折叠为“高级模式”，面向技术用户和管理员。

\- 后台表现：Sub2API 承担用户、套餐、余额、支付、兑换码、用量、账号池、限流、故障切换和管理员监控。

\- 运营闭环：用户购买套餐后自动获得 Codex++ 专属 API Key；余额不足时客户端提示续费；支付成功后即时恢复可用。

\- 差异化卖点：Codex++ 不改原 Codex 安装文件，通过 launcher 和 CDP 注入增强；Sub2API 负责稳定调度和计费，形成桌面端加云端服务的一体化产品。



\## 关键改造

\- Codex++ 客户端新增“账号登录绑定”流程：支持登录、设备绑定、套餐状态拉取、服务凭证自动写入、凭证轮换和一键修复。

\- Codex++ 默认创建一个托管供应商，例如 `Codex++ Cloud`，自动生成 Codex 运行配置；用户界面不展示真实 Key，只展示服务状态。

\- Sub2API 新增桌面客户端专用接口：`/api/v1/client/bootstrap` 返回网关地址、用户专属 API Key、默认模型、额度状态、公告和版本策略。

\- Sub2API 继续复用已有支付、余额、兑换码、API Key、用量统计、账号池调度、OpenAI/Codex Responses 网关能力。

\- 管理后台增加 Codex++ 产品维度：套餐配置、设备管理、客户端版本策略、用户服务状态、异常用户和用量排行。

\- 安全策略：下发的是 Sub2API 用户侧 Key，不是上游原始 Key；支持设备撤销、Key 轮换、余额不足停用、异常用量限流。



\## 对外接口与数据

\- `GET /api/v1/client/bootstrap`：客户端登录后拉取托管配置，返回 `gateway\_base\_url`、`api\_key`、`default\_model`、`wire\_api`、`quota`、`subscription`、`feature\_flags`。

\- `POST /api/v1/client/devices`：登记或刷新设备，字段包含 `device\_id`、`platform`、`app\_version`、`codex\_version`、`last\_seen\_at`。

\- `POST /api/v1/client/redeem`：可选兑换码激活，用于分销或补偿。

\- `GET /api/v1/client/usage`：返回余额、套餐、今日消耗、限流状态和续费入口。

\- 新增核心类型：`DesktopDevice`、`ClientBootstrapConfig`、`ManagedProviderProfile`、`ClientEntitlement`。



\## 里程碑

\- MVP：登录绑定、购买后自动配置、一键启动、余额展示、余额不足提示、管理员手动发放套餐。

\- V1：接入 Sub2API 内置支付，支持支付宝、微信、Stripe 或 EasyPay；支付成功后自动开通额度。

\- V1.5：设备管理、Key 自动轮换、异常用量风控、客户端公告、版本强更/灰度更新。

\- V2：团队套餐、企业发票、多人额度池、客服工单、模型包分层和白标分销。



\## 测试与验收

\- 新用户购买后首次打开 Codex++，无需填写 API 配置即可启动 Codex 并完成一次请求。

\- 余额不足、套餐过期、Key 被撤销时，客户端能给出清晰状态和续费入口。

\- 客户端重装或换设备后，登录同一账号能重新拉取托管配置。

\- Sub2API 支付成功后余额到账，Codex++ 在下一次刷新或启动时恢复可用。

\- 管理员能查看用户、设备、API Key、用量、支付订单和异常状态。

\- 现有高级用户仍可进入高级供应商配置，不破坏 Codex++ 原来的手动中转能力。



\## 假设

\- 默认采用“账号登录绑定”模式，不采用安装包内置密钥。

\- 用户不需要看到或复制 API Key；Key 只作为客户端到 Sub2API 网关的凭证。

\- 产品对外表达避免承诺“官方无限制”或规避条款，重点放在便捷配置、服务稳定、用量透明和合规自担。

\- 第一版优先做产品方案与体验闭环，暂不重构 Codex++ 现有供应商配置体系，只把它隐藏为高级能力。
